# Trading Journal Frontend

React приложение для просмотра и анализа журнала торговых позиций с фильтрацией, статистикой и визуализацией сделок.

## Основные возможности

- **Просмотр сделок**: Карточки с детальной информацией о каждой позиции
- **Фильтрация**: Множественные фильтры по символу, направлению, дате, статусу, источнику
- **Статистика**: Агрегированная статистика по всем позициям
- **Скриншоты**: Просмотр скриншотов графиков для каждой сделки
- **Удаление сделок**: Управление записями в журнале

## Конфигурация

### API Backend

По умолчанию приложение использует `http://localhost:3000/api` как базовый URL API.

Для изменения используйте переменную окружения:
```bash
VITE_API_BASE_URL=http://your-api-url/api npm run dev
```

## Основные функции

### Фильтрация

- **По символу**: Фильтрация по торговой паре (BTCUSDT, ETHUSDT и т.д.)
- **По направлению**: Long или Short позиции
- **По датам**: Диапазон дат создания сделок
- **По статусу**: Прибыльные/убыточные/все сделки
- **По источнику**: Тип источника данных (TVX и т.д.)
- **По сессии**: Фильтрация по торговой сессии

### Статистика

- Общее количество сделок
- Количество прибыльных/убыточных сделок
- Общая прибыль/убыток
- Процент прибыльных сделок
- Средняя прибыль/убыток на сделку

### Карточки сделок

Каждая карточка отображает:
- Торговая пара и таймфрейм
- Направление (Long/Short)
- Цены входа, стоп-лосса, тейк-профита
- Размер риска и потенциальная прибыль
- Даты создания и обновления
- Скриншоты графиков
- Дополнительная информация (TVX значения и т.д.)

### Просмотр скриншотов

- Модальное окно для просмотра скриншотов в полном размере
- Навигация между скриншотами одной сделки
- Zoom и pan для детального просмотра

## Интеграция с Backend

Приложение интегрируется с `trading-back` сервером для:
- Получения списка всех сделок
- Фильтрации сделок по различным параметрам
- Получения статистики
- Удаления сделок
- Получения скриншотов

## API Endpoints

- `GET /api/trades` - Получение списка сделок с фильтрами
- `GET /api/trades/stats` - Получение статистики
- `DELETE /api/trades/:id` - Удаление сделки
- `GET /api/screenshots/:filename` - Получение скриншота

## Разработка

Проект следует атомарной архитектуре:
- Каждый хук выполняет одну задачу
- Компоненты разделены по ответственности
- Стили вынесены в отдельные файлы
